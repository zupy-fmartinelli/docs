## Overview

This administrative endpoint allows authorized systems to add loyalty points directly to a customer's account with comprehensive audit logging. This is typically used for promotional adjustments, compensation, bonus campaigns, or manual corrections by customer service teams.

<Warning>
**Administrative Access Required**: This endpoint is designed for trusted systems and requires appropriate API key permissions. All transactions are logged with audit trails for compliance and tracking purposes.
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with administrative privileges
  ```
  Authorization: Bearer {your_admin_api_key}
  ```
</ParamField>

## Request Body

<ParamField body="customer_id" type="string" required>
  Zupy customer identifier (UUID format)
</ParamField>

<ParamField body="amount" type="integer" required>
  Number of points to add (must be positive integer)
</ParamField>

<ParamField body="reason" type="string">
  Human-readable reason for the points addition (default: "Administrative point addition")
</ParamField>

<ParamField body="admin_user" type="string">
  Email or identifier of the administrator performing the action
</ParamField>

<ParamField body="reference_id" type="string">
  Optional external reference ID for tracking (e.g., support ticket, promotion code)
</ParamField>

```python Python
import requests
import json

def add_customer_points(customer_id, amount, reason=None, admin_user=None, reference_id=None, api_key=None):
    """
    Add loyalty points to customer account
    """
    url = "https://api-staging.zupy.com/v1/loyalty/points/add"
    
    headers = {
        'Authorization': f'Bearer {api_key}',
        'Content-Type': 'application/json'
    }
    
    payload = {
        'customer_id': customer_id,
        'amount': amount,
        'reason': reason or 'Administrative point addition',
        'admin_user': admin_user,
        'reference_id': reference_id
    }
    
    try:
        response = requests.post(url, headers=headers, json=payload)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.HTTPError as e:
        error_detail = response.json() if response.content else {'error': str(e)}
        raise Exception(f"Failed to add points: {error_detail}")

# Usage example
result = add_customer_points(
    customer_id='cust_12345',
    amount=100,
    reason='Compensation for delayed order #ORD789',
    admin_user='support@restaurant.com',
    reference_id='TICKET-2025-001',
    api_key=admin_api_key
)

print(f"Added {result['amount_added']} points. New balance: {result['new_balance']}")
```

```php PHP
<?php
function addCustomerPoints($customerId, $amount, $reason = null, $adminUser = null, $referenceId = null, $apiKey) {
    $url = "https://api-staging.zupy.com/v1/loyalty/points/add";
    
    $headers = [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json'
    ];
    
    $payload = json_encode([
        'customer_id' => $customerId,
        'amount' => $amount,
        'reason' => $reason ?: 'Administrative point addition',
        'admin_user' => $adminUser,
        'reference_id' => $referenceId
    ]);
    
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL => $url,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_POST => true,
        CURLOPT_HTTPHEADER => $headers,
        CURLOPT_POSTFIELDS => $payload,
        CURLOPT_TIMEOUT => 30
    ]);
    
    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);
    
    if ($httpCode !== 200) {
        $error = json_decode($response, true);
        throw new Exception("Failed to add points: " . ($error['error']['message'] ?? 'Unknown error'));
    }
    
    return json_decode($response, true);
}

        $adminApiKey
    );
    
    echo "Added {$result['amount_added']} points. New balance: {$result['new_balance']}\n";
} catch (Exception $e) {
    echo "Error: " . $e->getMessage() . "\n";
}
?>
```

## Integration Best Practices

### Audit Trail Management

1. **Always Include Reason**: Provide clear, descriptive reasons for all point additions
2. **Reference IDs**: Use external reference IDs for tracking and correlation
3. **Admin Identification**: Always specify the administrator performing the action
4. **Documentation**: Maintain internal records linking to these transactions

### Error Handling

```javascript
const addPointsWithErrorHandling = async (pointsData) => {
  try {
    const result = await addPoints(pointsData);
    
    // Log successful operation
    console.log(`Successfully added ${result.amount_added} points to customer ${result.customer_id}`);
    
    // Update local systems
    await updateLocalCustomerBalance(result.customer_id, result.new_balance);
    
    return result;
    
  } catch (error) {
    // Handle specific error types
    if (error.message.includes('CUSTOMER_NOT_FOUND')) {
      // Customer might need to be created first
      console.warn('Customer not found, consider enrollment flow');
    } else if (error.message.includes('RATE_LIMIT_EXCEEDED')) {
      // Implement retry with backoff
      console.warn('Rate limited, implementing retry strategy');
      await new Promise(resolve => setTimeout(resolve, 60000)); // Wait 1 minute
      return addPointsWithErrorHandling(pointsData); // Retry
    } else if (error.message.includes('INSUFFICIENT_PERMISSIONS')) {
      // Handle permission issues
      console.error('API key lacks required permissions');
      throw new Error('Administrative access required');
    }
    
    throw error;
  }
};
```

### Bulk Operations

For adding points to multiple customers, make individual API calls with appropriate delays to respect rate limits:

```javascript
const addPointsBulk = async (operations) => {
  const results = [];
  
  for (const operation of operations) {
    try {
      const result = await addPoints(operation);
      results.push({ success: true, ...result });
      
      // Respect rate limits (10 operations per minute)
      await new Promise(resolve => setTimeout(resolve, 6000)); // 6 second delay
      
    } catch (error) {
      results.push({ 
        success: false, 
        customer_id: operation.customerId, 
        error: error.message 
      });
    }
  }
  
  return results;
};
```

### Security Considerations

- **API Key Security**: Use dedicated administrative API keys with restricted permissions
- **Audit Logging**: All operations are logged with IP addresses and timestamps
- **Amount Validation**: Validate point amounts on client side before API calls
- **Reference Tracking**: Always use reference IDs for accountability and tracking
