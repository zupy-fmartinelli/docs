## Overview

This administrative endpoint allows authorized systems to subtract loyalty points from a customer's account with comprehensive validation and audit logging. This is typically used for corrections, fraud prevention, expired points cleanup, or administrative adjustments by customer service teams.

<Warning>
**Administrative Access Required**: This endpoint requires administrative privileges and includes safeguards against creating negative balances unless explicitly allowed. All transactions are logged with complete audit trails.
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with administrative privileges
  ```
  Authorization: Bearer {your_admin_api_key}
  ```
</ParamField>

## Request Body

<ParamField body="customer_id" type="string" required>
  Zupy customer identifier (UUID format)
</ParamField>

<ParamField body="amount" type="integer" required>
  Number of points to subtract (must be positive integer)
</ParamField>

<ParamField body="reason" type="string">
  Human-readable reason for the points subtraction (default: "Administrative point deduction")
</ParamField>

<ParamField body="admin_user" type="string">
  Email or identifier of the administrator performing the action
</ParamField>

<ParamField body="reference_id" type="string">
  Optional external reference ID for tracking (e.g., support ticket, fraud case)
</ParamField>

<ParamField body="allow_negative" type="boolean">
  Whether to allow the operation to result in negative balance (default: false)
</ParamField>

```python Python
import requests
import json

def subtract_customer_points(customer_id, amount, reason=None, admin_user=None, 
                           reference_id=None, allow_negative=False, api_key=None):
    """
    Subtract loyalty points from customer account
    """
    url = "https://api-staging.zupy.com/v1/loyalty/points/subtract"
    
    headers = {
        'Authorization': f'Bearer {api_key}',
        'Content-Type': 'application/json'
    }
    
    payload = {
        'customer_id': customer_id,
        'amount': amount,
        'reason': reason or 'Administrative point deduction',
        'admin_user': admin_user,
        'reference_id': reference_id,
        'allow_negative': allow_negative
    }
    
    try:
        response = requests.post(url, headers=headers, json=payload)
        response.raise_for_status()
        return response.json()
    except requests.exceptions.HTTPError as e:
        error_detail = response.json() if response.content else {'error': str(e)}
        raise Exception(f"Failed to subtract points: {error_detail}")

# Usage example - expired points cleanup
result = subtract_customer_points(
    customer_id='cust_12345',
    amount=75,
    reason='Expired points cleanup - points older than 12 months',
    admin_user='system@restaurant.com',
    reference_id='EXPIRY-CLEANUP-2025-Q1',
    allow_negative=False,
    api_key=admin_api_key
)

print(f"Subtracted {result['amount_subtracted']} points. New balance: {result['new_balance']}")
```

```php PHP
<?php
function subtractCustomerPoints($customerId, $amount, $reason = null, $adminUser = null, 
                               $referenceId = null, $allowNegative = false, $apiKey) {
    $url = "https://api-staging.zupy.com/v1/loyalty/points/subtract";
    
    $headers = [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json'
    ];
    
    $payload = json_encode([
        'customer_id' => $customerId,
        'amount' => $amount,
        'reason' => $reason ?: 'Administrative point deduction',
        'admin_user' => $adminUser,
        'reference_id' => $referenceId,
        'allow_negative' => $allowNegative
    ]);
    
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL => $url,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_POST => true,
        CURLOPT_HTTPHEADER => $headers,
        CURLOPT_POSTFIELDS => $payload,
        CURLOPT_TIMEOUT => 30
    ]);
    
    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);
    
    if ($httpCode !== 200) {
        $error = json_decode($response, true);
        throw new Exception("Failed to subtract points: " . ($error['error']['message'] ?? 'Unknown error'));
    }
    
    return json_decode($response, true);
}

} catch (Exception $e) {
```

## Integration Best Practices

### Balance Validation

Always check customer balance before large subtractions to avoid errors:

```javascript
const safeSubtractPoints = async (customerId, amount, reason, adminUser) => {
  // First check current balance
  const balance = await getCustomerBalance(customerId);
  
  if (balance.current_balance < amount) {
    console.warn(`Insufficient balance: ${balance.current_balance} < ${amount}`);
    
    // Option 1: Subtract only available points
    const availableAmount = balance.current_balance;
    if (availableAmount > 0) {
      return await subtractPoints({
        customerId,
        amount: availableAmount,
        reason: `${reason} (partial - insufficient balance)`,
        adminUser
      });
    }
    
    // Option 2: Allow negative balance if business rules permit
    // return await subtractPoints({
    //   customerId,
    //   amount,
    //   reason,
    //   adminUser,
    //   allowNegative: true
    // });
    
    throw new Error('Insufficient balance for subtraction');
  }
  
  return await subtractPoints({
    customerId,
    amount,
    reason,
    adminUser
  });
};
```

### Error Handling with Retry Logic

```javascript
const subtractPointsWithRetry = async (pointsData, maxRetries = 3) => {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const result = await subtractPoints(pointsData);
      return result;
      
    } catch (error) {
      if (error.message.includes('INSUFFICIENT_BALANCE')) {
        // Don't retry balance issues
        throw error;
      }
      
      if (error.message.includes('RATE_LIMIT_EXCEEDED')) {
        if (attempt < maxRetries) {
          const delay = Math.pow(2, attempt) * 1000; // Exponential backoff
          console.log(`Rate limited, retrying in ${delay}ms...`);
          await new Promise(resolve => setTimeout(resolve, delay));
          continue;
        }
      }
      
      if (attempt === maxRetries) {
        throw error;
      }
      
      // General retry delay
      await new Promise(resolve => setTimeout(resolve, 1000));
    }
  }
};
```

### Audit Trail Best Practices

1. **Detailed Reasons**: Always provide specific, actionable reasons
2. **Reference IDs**: Link to external systems (tickets, cases, orders)  
3. **Admin Identification**: Track who performed the operation
4. **Business Justification**: Include business context in reasons

### Security Considerations

- **Double Authorization**: Consider requiring two-factor authentication for large amounts
- **Amount Limits**: Implement daily/monthly limits for point subtractions
- **Fraud Detection**: Monitor patterns of large subtractions
- **Balance Protection**: Use `allow_negative: false` by default to prevent accidental overdrafts
