## Overview

This endpoint returns the complete catalog of rewards available in your loyalty program. Customers can redeem their earned points for these rewards. The catalog includes food items, drinks, discounts, and special experiences.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication
  ```
  Authorization: Bearer {your_api_key}
  ```
</ParamField>

## Path Parameters

<ParamField path="program_id" type="string" required>
  Your loyalty program identifier (provided during onboarding)
  ```
  prog_bella_vista
  ```
</ParamField>

## Query Parameters

<ParamField query="category" type="string">
  Filter rewards by category
  ```
  food | drinks | discounts | experiences
  ```
</ParamField>

<ParamField query="min_points" type="number">
  Filter rewards requiring at least this many points
  ```
  100
  ```
</ParamField>

<ParamField query="max_points" type="number">
  Filter rewards requiring at most this many points
  ```
  1000
  ```
</ParamField>

<ParamField query="available_only" type="boolean">
  Show only rewards currently available for redemption (default: true)
  ```
  true | false
  ```
</ParamField>

<ParamField query="limit" type="number">
  Maximum number of rewards to return (default: 50)
  ```
  20
  ```
</ParamField>

## Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.zupy.com/v1/loyalty/programs/prog_bella_vista/rewards?category=food&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://api.zupy.com/v1/loyalty/programs/prog_bella_vista/rewards?category=food&limit=10', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

url = "https://api.zupy.com/v1/loyalty/programs/prog_bella_vista/rewards"
params = {
    "category": "food",
    "limit": 10
}
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

response = requests.get(url, params=params, headers=headers)
data = response.json()
print(data)
```

```php PHP
<?php
$program_id = "prog_bella_vista";
$url = "https://api.zupy.com/v1/loyalty/programs/{$program_id}/rewards?" . http_build_query([
    'category' => 'food',
    'limit' => 10
]);

$headers = [
    "Authorization: Bearer YOUR_API_KEY",
    "Content-Type: application/json"
];

$response = file_get_contents($url, false, stream_context_create([
    'http' => [
        'method' => 'GET',
        'header' => implode("\r\n", $headers)
    ]
]));

$data = json_decode($response, true);
print_r($data);
?>
```
</CodeGroup>

## Reward Availability

<Note>
  **Stock Management**: Limited rewards show `stock_count` and are automatically hidden when out of stock unless `available_only=false` is specified.
</Note>

<Warning>
  **Expiration**: Time-limited rewards include `valid_until` timestamp. Expired rewards are automatically filtered out unless explicitly requested.
</Warning>

## Dietary Information

For food and drink rewards, detailed dietary information is provided:

- **Vegetarian/Vegan**: Plant-based options clearly marked
- **Gluten-Free**: Celiac-safe options identified  
- **Allergens**: Complete allergen list for safety
- **Nutritional Info**: Available in extended response (add `?include=nutrition`)

## Integration Tips

### Display Recommendations
1. **Sort by Points**: Show achievable rewards first based on customer's current points
2. **Visual Categories**: Use category icons and colors for better UX
3. **Progress Indicators**: Show how close customers are to earning each reward
4. **Stock Alerts**: Display limited availability prominently

### Customer Experience
1. **Personalization**: Filter based on customer preferences and dietary restrictions
2. **Recommendations**: Use purchase history to suggest relevant rewards
3. **Motivation**: Show "next achievable reward" to encourage engagement

## Rate Limits

- **Production**: 1000 requests per minute
- **Staging**: 100 requests per minute
