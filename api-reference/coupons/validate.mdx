## Overview

This endpoint validates coupon codes before usage, returning detailed discount information and eligibility status. It performs comprehensive validation including expiration dates, usage limits, order restrictions, and customer eligibility rules.

<Note>
  **Integration Flow**: Use this endpoint before order finalization to check coupon validity and calculate discounts. Follow with the [Use Coupon](/api-reference/coupons/redeem) endpoint after successful payment.
</Note>

## Discount Types Explained

### Percentage Discount
Calculates discount as percentage of order total:
```
discount_amount = (order_total * discount_percentage) / 100
final_total = order_total - discount_amount
```

**Maximum Discount Cap**: Some percentage coupons have maximum discount limits to prevent excessive discounts on large orders.

### Fixed Amount Discount  
Applies fixed monetary discount:
```
discount_amount = fixed_discount_value
final_total = max(0, order_total - discount_amount)
```

**Minimum Order Protection**: Fixed discounts usually require minimum order amounts to prevent negative totals.

### Free Item Discount
Adds specific item to order at no charge:
- Item is identified by `free_item_id` (SKU/product code)
- Item must be added to cart by the integration system
- `discount_amount` is typically "0.00" as item addition is handled separately

## Business Rules Validation

### Order Type Restrictions
Coupons can be restricted to specific order types:

<ResponseField name="DELIVERY">
  **Delivery Orders**: Orders delivered to customer address
</ResponseField>

<ResponseField name="PICKUP">
  **Pickup Orders**: Customer collects order at store
</ResponseField>

<ResponseField name="DINE_IN">
  **Dine-in Orders**: Customer eats at restaurant
</ResponseField>

### Customer Eligibility
- **First-time Customer**: Some coupons only for new customers
- **Loyalty Tier**: Certain coupons restricted by customer tier (Bronze, Silver, Gold, etc.)
- **Usage History**: Limits on how many times each customer can use specific coupons

### Item Restrictions
- **Excluded Items**: Some items cannot be discounted (e.g., delivery fees, alcohol)
- **Category Restrictions**: Discounts may apply only to specific product categories
- **FREE_ITEM Availability**: Free items must be in stock and available

## Error Codes

| Code | Description | Action Required |
|------|-------------|-----------------|
| `COUPON_EXPIRED` | Coupon has passed expiration date | Show expiration date to customer |
| `COUPON_NOT_FOUND` | Invalid or inactive coupon code | Verify code spelling |
| `MINIMUM_ORDER_NOT_MET` | Order total below required minimum | Display required minimum amount |
| `USAGE_LIMIT_EXCEEDED` | Customer has used coupon maximum times | Explain usage limit |
| `INVALID_ORDER_TYPE` | Order type not allowed for coupon | Show valid order types |
| `CUSTOMER_NOT_ELIGIBLE` | Customer doesn't meet coupon requirements | Explain eligibility criteria |
| `FREE_ITEM_UNAVAILABLE` | Free item is out of stock | Offer alternative or rain check |
| `INVALID_CUSTOMER_ID` | Customer ID not found or invalid | Verify customer exists |

## Integration Best Practices

### Pre-validation Checks
Before calling the validation endpoint, verify:

1. **Coupon Code Format**: Basic format validation to avoid unnecessary API calls
2. **Customer Authentication**: Ensure customer is logged in and ID is available
3. **Order Completeness**: All required order data is collected

### Error Handling Strategy

<CodeGroup>
```javascript Error Handling Example
const validateCouponSafely = async (couponCode, orderData) => {
  try {
    const result = await validateCoupon(couponCode, orderData);
    
    if (result.valid) {
      // Apply discount to UI
      return {
        success: true,
        discount: result.discount,
        message: result.usage_instructions
      };
    } else {
      // Handle business rule violations gracefully
      const error = result.error;
      
      switch (error.code) {
        case 'MINIMUM_ORDER_NOT_MET':
          return {
            success: false,
            message: `Adicione mais R$ ${error.details.difference} para usar este cupom`,
            suggestedAction: 'increase_order'
          };
          
        case 'COUPON_EXPIRED':
          return {
            success: false,
            message: 'Este cupom expirou. Confira outros cupons disponíveis.',
            suggestedAction: 'show_available_coupons'
          };
          
        case 'USAGE_LIMIT_EXCEEDED':
          return {
            success: false,
            message: 'Você já usou este cupom. Confira outros cupons disponíveis.',
            suggestedAction: 'show_available_coupons'
          };
          
        default:
          return {
            success: false,
            message: error.message || 'Cupom inválido',
            suggestedAction: 'remove_coupon'
          };
      }
    }
    
  } catch (networkError) {
    // Handle network/API errors
    console.error('Coupon validation error:', networkError);
    return {
      success: false,
      message: 'Erro ao validar cupom. Tente novamente.',
      suggestedAction: 'retry'
    };
  }
};
```
</CodeGroup>

### Performance Optimization

- **Debounce Validation**: Avoid validating on every keystroke
- **Cache Results**: Cache validation results for 30-60 seconds
- **Async Validation**: Don't block UI while validating
- **Fallback Handling**: Gracefully handle API timeouts

## Rate Limits

- **Production**: 500 validations per minute per API key
- **Staging**: 100 validations per minute per API key

<Tip>
  **Performance**: Validation typically responds within 200ms. Cache results temporarily to avoid redundant validations for the same coupon + order combination.
</Tip>

<Warning>
  **Security**: Always validate coupons server-side before applying discounts. Never trust client-side validation alone for discount calculations.
</Warning>
