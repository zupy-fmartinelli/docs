## Overview

This endpoint returns a paginated list of all customers registered in the loyalty program. It supports pagination to handle large customer databases efficiently.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication
  ```
  Authorization: Bearer {your_api_key}
  ```
</ParamField>

## Query Parameters

<ParamField query="limit" type="integer" default="20">
  Number of results per page (max 100)
  ```
  20
  ```
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip for pagination
  ```
  0
  ```
</ParamField>

<ParamField query="ordering" type="string">
  Field to order results by. Prefix with `-` for descending order
  ```
  -created_at
  ```
  
  Available fields: `name`, `created_at`, `last_activity_date`, `points_balance`
</ParamField>

## Response

<ResponseField name="count" type="integer">
  Total number of customers
</ResponseField>

<ResponseField name="next" type="string">
  URL for the next page of results (null if last page)
</ResponseField>

<ResponseField name="previous" type="string">
  URL for the previous page of results (null if first page)
</ResponseField>

<ResponseField name="results" type="array">
  Array of customer objects

      Unique customer identifier
    </ResponseField>
    
    <ResponseField name="card_number" type="string">

    </ResponseField>
    
    <ResponseField name="user" type="object">
      Customer personal information
      
      <Expandable title="user object">
        <ResponseField name="name" type="string">
          Customer full name

        <ResponseField name="email" type="string">
          Customer email address
        </ResponseField>
        
        <ResponseField name="whatsapp_number" type="string">

        <ResponseField name="cpf" type="string">
          Customer CPF (Brazilian tax ID)
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="points_balance" type="integer">
      Current available points balance
    </ResponseField>
    
    <ResponseField name="points_earned" type="integer">

    </ResponseField>

      Total points spent/redeemed

      Customer loyalty tier/segment
    </ResponseField>
    
    <ResponseField name="active" type="boolean">
      Whether the customer account is active

    <ResponseField name="created_at" type="string">
      Account creation timestamp (ISO 8601)
    </ResponseField>
    
    <ResponseField name="last_activity_date" type="string">
      Last customer activity timestamp (ISO 8601)

    <ResponseField name="transaction_count" type="integer">
      Total number of transactions
    </ResponseField>
  </Expandable>
</ResponseField>

## Use Cases

- **Customer Management**: Administrative interface to browse all customers
- **Data Analytics**: Export customer data for analysis and reporting
- **Customer Service**: Browse customer list for support operations
- **Marketing Campaigns**: Build customer segments for targeted campaigns

## Pagination

This endpoint supports cursor-based pagination. Use the `next` and `previous` URLs to navigate through pages, or construct your own using `limit` and `offset` parameters.

Example pagination:
- Page 1: `limit=20&offset=0`
- Page 2: `limit=20&offset=20`  
- Page 3: `limit=20&offset=40`
