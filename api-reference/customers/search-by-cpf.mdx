## Overview

This endpoint provides a dedicated search for customers using their CPF (Brazilian tax ID). It's optimized for Point of Sale (POS) integration with enhanced data security features including CPF masking and sanitized customer data.

<Note>
  **Epic 2 Enhancement (Story 2.11.1)**: Dedicated CPF search endpoint with improved privacy controls and POS optimization.
</Note>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication
  ```
  Authorization: Bearer {your_api_key}
  ```
</ParamField>

## Path Parameters

<ParamField path="cpf" type="string" required>
  Customer's CPF (Brazilian tax ID) - 11 digits
  ```
  12345678901
  ```
</ParamField>

## Features

### Enhanced Privacy
- **CPF Masking**: CPF is automatically masked in response for privacy protection
- **Sanitized Data**: All sensitive data is processed and sanitized before response
- **Secure Lookup**: Direct CPF lookup without exposing full numbers in logs

### POS Integration Optimization
- **Fast Response**: Optimized for point-of-sale systems requiring quick customer lookup
- **Complete Profile**: Returns all necessary customer information in single request
- **Balance Information**: Immediate access to current points balance and tier status

## Use Cases

- **Point of Sale Integration**: Quick customer lookup during checkout process
- **Customer Service**: Support agents can find customers using CPF for assistance
- **Mobile App Integration**: Customer verification during app registration/login
- **Loyalty Card Replacement**: Verify customer identity when replacing physical cards

<Note>
  This endpoint is specifically designed for Brazilian market requirements where CPF is the primary customer identifier for loyalty programs and POS systems.
</Note>
